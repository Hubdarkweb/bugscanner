# BugScanner Web

A modern, fast, and Termux-compatible web interface for the BugScanner core reconnaissance tool.

![Screenshot](https://via.placeholder.com/800x450.png?text=BugScanner+Web+Interface) *(Screenshot placeholder)*

BugScanner Web wraps a customized Python reconnaissance script into an elegant web application reminiscent of Zenmap. The UI features stunning glassmorphism styles, fluid animations, and utilizes Server-Sent Events (SSE) to stream live terminal output back to your browser without hanging or reloading.

## ✨ Features

- **Real-time Terminal Output:** Commands execute securely in the background while standard output is streamed instantly to the UI terminal box.
- **Termux Ready:** Fully supports Android environments via Termux. Includes an auto-setup script for Linux/Termux `pkg` and Python dependencies.
- **Glassmorphism Design:** A beautiful dark-themed UI that is responsive on mobile and desktop.
- **Multiple Scan Modes:** 
  - Direct Scan
  - Proxy Scan
  - SSL Vulnerability Checking
  - UDP Port Sweeping
  - WebSocket Pings
  - Standard ICMP Ping

## 📦 Requirements

- Python 3.x
- Flask
- WebSocket-Client
- Multithreading
- Loguru
- Requests

## 🚀 Installation & Setup

### Automated (Termux & Linux)

The easiest way to set up BugScanner Web is to use the provided installer script.

1. Clone or copy the source code into a directory on your device.
2. Make the installer executable and run it:
   ```bash
   chmod +x install.sh
   ./install.sh
   ```
   *Note: If you run into Debian's externally-managed-environment issues on newer Termux systems, you may want to set up a virtual environment first or run `python3 -m venv venv && source venv/bin/activate` before installing requirements manually.*

### Manual

If you prefer to install the requirements manually:

1. Setup a virtual environment (optional but recommended):
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```
2. Install the requirements:
   ```bash
   pip install -r requirements.txt
   ```

## 💻 Usage

1. Start the web server:
   ```bash
   python3 app.py
   ```
2. Open your web browser and navigate to:
   ```
   http://127.0.0.1:5000
   ```
3. Enter your Target/CIDR range, select your preferred scan Profile, and hit **Start Scan**.

## ⚙️ How it Works

1. **Flask Backend:** `app.py` sets up a lightweight HTTP server.
2. **Subprocess execution:** When a scan is launched, `runner.py` kicks off the custom `scanner.py` script as an unbuffered subprocess.
3. **SSE Streaming:** Output lines generated by the script are caught by `runner.py` and yielded immediately via Server-Sent Events to `main.js` on the frontend.
4. **Rendering:** `main.js` processes these events and updates the web terminal window in real-time.
